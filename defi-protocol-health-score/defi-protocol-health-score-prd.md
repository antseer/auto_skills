# DeFi 平台/协议健康度综合评分器 — 需求文档

## 需求概述

用户在 DeFi 协议存入资金或在去中心化交易平台开仓时，几乎没有工具评估"平台本身安不安全"。Hyperliquid 上 $100M 清算事件、Aave/Chaos Labs 预言机配置错误导致 $25M 错误清算、小交易所交易量造假——这些风险事件反复证明平台级风险被严重低估。用户需要一个事前而非事后的风险评估工具。

需要一个通过链上资金流向、市场情绪、代币持仓集中度和历史异常事件分析，输出平台级别综合健康度评分的工具。

## 用户画像

### Persona 1: Carol — DeFi 收益策略研究员
- **角色**: 全职 DeFi 分析师，在多个协议有六位数存款
- **日常工作流**: 评估协议收益率 → 存入资金 → 定期监控
- **工具栈**: Python, Dune Analytics, DefiLlama API, Claude Code
- **痛点**: 在 Aave 有六位数存款但无法评估治理健康度，Chaos Labs 预言机错误导致 $25M 清算暴露了她完全忽视的风险维度

### Persona 2: Dave — 加密期权/衍生品交易员
- **角色**: 活跃衍生品交易者，在中心化和去中心化平台间分配仓位
- **日常工作流**: 监控 IV/Greeks → 选择执行平台 → 开仓 → 对冲
- **工具栈**: Python, Deribit API, TradingView, Claude Code
- **痛点**: James Wynn 在 Hyperliquid 被清算 $100M 暴露了去中心化衍生品平台的系统性风险，但他无法量化这种风险来做平台选择

### Persona 3: 链上数据散户（临时 Persona，源自 cluster_3）
- **角色**: 散户投资者，曾在小交易所踩坑
- **日常工作流**: 搜索新代币 → 寻找上市交易所 → 注册 → 交易
- **工具栈**: Glassnode, Dune, Claude Code, Excel
- **痛点**: 在小交易所（FameEX）买过山寨币，后知道交易量造假。Zonda 用户连提币都提不出来。

## 场景表

| 场景 | 触发条件 | 期望产出 |
|------|----------|----------|
| 大额存款前评估 | 考虑向 DeFi 协议存入资金之前 | 综合健康度评分 + 风险维度明细 |
| 衍生品平台选择 | 选择衍生品交易执行平台时 | 去中心化 vs 中心化平台风险对比 |
| 新交易所注册前 | 在新交易所注册或充值之前 | 简单安全评分（类似信用评级） |
| 定期健康审查 | 定期审查已有持仓所在平台 | 健康状态变化追踪 |

## 用户故事

1. **As Carol**, I want 在向 DeFi 协议存入大额资金前自动获取该协议的治理风险、预言机依赖度和历史异常事件汇总, so that 我能在存款决策中纳入平台级风险因子而非只看收益率。
2. **As Dave**, I want 对比去中心化和中心化衍生品平台的系统性风险指标, so that 我能根据风险偏好和仓位大小选择最合适的执行平台。
3. **As 散户投资者**, I want 在新交易所注册前获取一个简单的安全评分（类似信用评级）, so that 我不再因为追逐只在小交易所上市的代币而承担平台跑路风险。

## 讨论引言

该需求源自讨论中三类用户对 DeFi/交易所平台级风险的共同关注。Carol 从 Aave/Chaos Labs 事件中意识到收益率分析忽视了平台本身的风险维度；Dave 从 Hyperliquid 清算事件认识到去中心化衍生品平台的系统性风险需要量化；散户投资者则从小交易所踩坑经历中渴望事前风险评估。三者共同指向"缺乏平台级事前风险评估工具"这一空白。

## 需求边界

### 包含
- 资金流向健康度评估（鲸鱼占比、资金净流向、聪明钱持仓）
- 市场结构健康度评估（订单簿深度、波动率、机构兴趣）
- 衍生品市场健康度评估（OI 集中度、资金费率、多空比）
- 社区/叙事健康度评估（情绪分析、风险关键词、讨论多样性）
- 四维度综合评分（0-100）
- 同类协议对比
- 适用性建议（大额存款/杠杆交易/长期持仓）

### 不包含
- 智能合约代码审计
- DAO 治理提案详细分析
- 具体预言机节点运行状态
- 对无衍生品数据的协议强行评分衍生品维度
