# 宏观事件→DeFi 影响传导链评估器 — 需求文档

## 需求概述

重大地缘政治或宏观经济事件（如中东冲突升级、央行利率决议、监管法案通过）对加密市场和 DeFi 生态有多层传导效应。但用户目前需要在 5+ 个工具间手动切换（新闻源→商品行情→链上 TVL→借贷利率→stablecoin 流向），完成一次完整评估需 1-3 小时，往往滞后于市场反应。

需要一个自动构建从事件到链上数据的多层传导链工具，在 15 分钟内输出结构化的事件影响评估报告。

## 用户画像

### Persona 1: Carol — DeFi 收益策略研究员
- **角色**: 全职 DeFi 收益分析师，每天用 Python + DefiLlama + Dune 评估协议收益率
- **日常工作流**: 抓取 DeFi 协议数据 → 对比收益率 → 评估风险 → 撰写策略报告
- **工具栈**: Python, Dune Analytics, DefiLlama API, Claude Code, Jupyter Notebook
- **痛点**: 伊朗宣布霍尔木兹管控后花 2 小时在 TradingView/DefiLlama/Dune/Aave 间切换做传导分析，策略报告总是滞后于市场反应

### Persona 2: 宏观地缘分析师（临时 Persona，源自 cluster_1）
- **角色**: 地缘政治敏感型宏观交易者，追踪中东局势和原油期货
- **日常工作流**: 追踪地缘新闻 → 构建传导链 → 评估投资组合影响
- **工具栈**: Python, TradingView, Bloomberg 替代方案, Claude Code
- **痛点**: 每次地缘新闻后需 3 小时手动构建传导链，到完成时市场已消化大部分信息

### Persona 3: 链上数据散户（临时 Persona，源自 cluster_3）
- **角色**: 持仓驱动型散户投资者，持有 XRP/BTC/ETH
- **日常工作流**: 查链上数据 → 搜监管新闻 → 估算影响
- **工具栈**: Glassnode, Dune Analytics, Claude Code, Excel
- **痛点**: 看到伊朗新闻很慌但不知道对自己持仓的具体影响，什么都没做结果 XRP 跌了 8%

## 场景表

| 场景 | 触发条件 | 期望产出 |
|------|----------|----------|
| 重大地缘政治新闻 | 军事冲突、外交谈判破裂、贸易制裁 | 事件→商品→通胀→利率→BTC/ETH→DeFi 的完整传导链评估 |
| 央行利率决议 | 利率决议或通胀数据发布 | 宏观指标变化对 DeFi 借贷利率和 TVL 的影响评估 |
| 监管法案通过 | 重大执法行动或法案通过 | 监管对链上资金流向和 DeFi 生态的影响评估 |
| 快速持仓影响评估 | 用户需要评估宏观事件对 DeFi 策略或持仓的影响 | 结构化多层传导链报告 |

## 用户故事

1. **As Carol**, I want 输入地缘事件关键词后自动获取从商品价格到 DeFi 协议 TVL/利率的完整传导链量化评估, so that 我能在 15 分钟内（而非 2 小时）判断事件对我推荐的收益策略的影响。
2. **As 宏观分析师**, I want 一键生成事件→原油→通胀→利率→BTC/ETH 的多层传导链报告, so that 我能在市场消化信息前完成投资组合影响评估。
3. **As 散户投资者**, I want 用简单语言获知重大新闻对我持仓币种的具体影响百分比估计, so that 我不再因为信息焦虑而错过设置止损的时机。

## 讨论引言

该需求源自 auto_req 流水线讨论中多位 persona 的共识: 地缘/宏观事件对 DeFi 生态的影响评估目前高度碎片化，用户需要跨越新闻源、商品行情、链上数据、DeFi 协议指标等多个层次才能完成一次评估。Carol 指出其策略报告因手动流程总是滞后于市场反应；宏观分析师强调传导链构建的手动耗时问题；散户投资者则因无法快速理解事件影响而产生信息焦虑和错误决策。

## 需求边界

### 包含
- 地缘/宏观事件识别与市场情绪获取
- 宏观经济基准数据聚合（CPI、利率、国债收益率）
- 链上资金流向追踪（BTC/ETH/stablecoin）
- 市场技术指标获取和分析
- DeFi 协议相关话题热度追踪
- 多层传导链构建与结构化报告生成

### 不包含
- 期权定价
- 杠杆倍数建议
- 具体协议智能合约风险评估
- 方向性交易建议
