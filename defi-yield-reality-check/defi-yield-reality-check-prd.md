# DeFi 资金池真实净收益率拆解器 — 需求文档

## 需求概述

DeFi 协议普遍以聚合 APY 作为营销数字，但表面 APY 与用户实际可获得的净收益之间存在系统性偏差。常见陷阱包括：代币激励占 APY 60%+ 且代币正在通胀稀释、无常损失在波动市中吞掉大部分收益、Gas 费累计侵蚀小额头寸的回报。用户每次拆解一个池子需 2-3 小时手动计算。

需要一个自动拆解表面 APY 为各组成部分，估算隐性成本（代币激励稀释、Gas、无常损失风险），输出风险调整后的净收益率评估工具。

## 用户画像

### Persona 1: Carol — DeFi 收益策略研究员
- **角色**: 全职 DeFi 分析师，每天评估多个协议的收益策略
- **日常工作流**: DefiLlama 查 TVL/APY → Dune 查链上交易量和大户持仓 → 手动计算扣除无常损失后的净收益率
- **工具栈**: Python, Dune Analytics, DefiLlama API, Claude Code, Jupyter Notebook
- **痛点**: 差点推荐 Arbitrum 上宣称 85% APY 的池子，3 小时分析后发现真实净年化仅约 7%。每周都有高 APY 宣传需要拆解，每次耗时 2-3 小时。

### Persona 2: 链上数据散户（临时 Persona，源自 cluster_3）
- **角色**: 持仓驱动型散户投资者
- **日常工作流**: 在社交媒体看到 DePIN/DeFi 项目收益宣传 → 研究项目 → 投资决策
- **工具栈**: Glassnode, Dune Analytics, Claude Code, Excel
- **痛点**: 因看到 DePIN 项目月收入 $500K 而买入代币，后发现 80% 是代币激励而非真实用户付费。缺乏识别"面值 vs 实值"偏差的能力。

## 场景表

| 场景 | 触发条件 | 期望产出 |
|------|----------|----------|
| 高 APY 宣传验证 | 看到 DeFi 协议或资金池的高 APY 宣传 | APY 完整组成拆解 + 风险调整后净收益率 |
| 多协议收益率对比 | 需要选择最优 DeFi 收益策略 | 同赛道协议的净 APY 对比表 |
| DePIN 收入验证 | DePIN 项目声称高收入 | "真实用户付费 vs 代币激励"比例分析 |
| 已投头寸回报评估 | 定期评估已投入的 DeFi 头寸 | 当前实际回报 vs 宣传回报对比 |

## 用户故事

1. **As Carol**, I want 输入 DeFi 资金池地址后自动获取 APY 的完整组成拆解（基础交易费、代币激励、借贷利息）及各项隐性成本, so that 我能在 10 分钟内判断一个池子的真实净收益率而非花 3 小时手动计算。
2. **As 散户投资者**, I want 在看到高收益宣传时能一键查看"宣传 APY vs 真实净 APY"的对比, so that 我不再因表面数字误导而做出亏损投资。

## 讨论引言

该需求源自 Carol 和散户投资者在讨论中的核心痛点反馈。Carol 指出 DeFi 收益率拆解是她最耗时的日常工作之一，特别是代币激励稀释和无常损失的量化计算；散户投资者则因缺乏拆解能力而多次因表面 APY 误导而亏损。两者共同指向"表面数字 vs 真实收益"的认知鸿沟。

## 需求边界

### 包含
- APY 组成拆解（基础交易费、代币激励、借贷利息）
- 代币激励稀释风险评估（基于代币价格趋势和交易所流向）
- 无常损失估算（基于波动率指标）
- Gas 成本估算
- 同赛道协议对比
- 风险调整后净 APY 计算

### 不包含
- 非 AMM 类 DeFi 协议拆解逻辑（纯借贷如 Aave 存款无无常损失）
- 池级 APY 组件实时数据（需 DefiLlama/Dune，当前 MCP 不提供）
- 投资建议
