# auto-req-output: Synthesis and Skill Generation

Read all discussion rounds, extract requirements, rank them, and generate skill drafts.

## Setup

Get today's date (YYYY-MM-DD). Load these files:
- `discussions/YYYY-MM-DD/round1.json`
- `discussions/YYYY-MM-DD/round2.json`
- `discussions/YYYY-MM-DD/round3.json`
- `raw_posts/YYYY-MM-DD.json` (fallback: `test_fixtures/raw_posts_sample.json`)

If any discussion file is missing, print an error and stop: "Missing: discussions/YYYY-MM-DD/roundN.json — run /auto-req-simulate first."

## Step 1: Synthesize Requirements

Dispatch a single subagent with this prompt (replace [RAW_POSTS_JSON], [ROUND1_JSON], [ROUND2_JSON], [ROUND3_JSON] with actual file contents):

```
REQUIREMENTS SYNTHESIZER

You have observed a 3-round discussion between 4 developer personas reacting to real Claude Code posts on X/Twitter.

RAW POSTS (what users actually said publicly):
[RAW_POSTS_JSON]

ROUND 1 (first persona reactions):
[ROUND1_JSON]

ROUND 2 (cross-reactions, new realizations):
[ROUND2_JSON]

ROUND 3 (facilitator themes + persona probe responses):
[ROUND3_JSON]

Extract ALL requirements — both explicit (directly stated in posts) and implicit (emerged from discussion dynamics, not stated in original posts).

For each requirement, score "resonance" as X/4 where X = number of personas who independently raised or strongly agreed with this need.

Output a JSON object (no other text):
{
  "explicit_needs": [
    {
      "need": "clear one-line description",
      "resonance": "3/4",
      "source_posts": ["post_001"],
      "representative_quote": "exact quote from a post or persona"
    }
  ],
  "implicit_needs": [
    {
      "need": "clear one-line description",
      "resonance": "4/4",
      "emerged_in": "Round 2 cross-reaction / Round 3 theme probe",
      "why_implicit": "why this didn't appear in the original posts",
      "representative_quote": "exact quote from discussion"
    }
  ],
  "priority_ranking": [
    {
      "rank": 1,
      "need": "same text as above",
      "type": "explicit / implicit",
      "resonance": "4/4",
      "rationale": "why this ranks #1"
    },
    { "rank": 2, "need": "...", "type": "...", "resonance": "...", "rationale": "..." },
    { "rank": 3, "need": "...", "type": "...", "resonance": "...", "rationale": "..." }
  ]
}
```

## Step 2: Write Requirements Report

Using the synthesizer output, write `reports/YYYY-MM-DD-requirements.md`:

```markdown
# Claude Code Skill Requirements — YYYY-MM-DD

> Auto-generated by auto-req pipeline. Source: [N] X/Twitter posts, 4 simulated personas, 3 discussion rounds.

## Explicit Needs
*(Directly stated in posts)*

| # | Need | Resonance | Source |
|---|------|-----------|--------|
| 1 | ... | 3/4 | [post](url) |

## Implicit Needs
*(Emerged from discussion, not stated in original posts)*

| # | Need | Resonance | Emerged In |
|---|------|-----------|------------|
| 1 | ... | 4/4 | Round 2 |

## Priority Ranking

1. **[Need]** (resonance: 4/4) — [rationale]
2. **[Need]** (resonance: 3/4) — [rationale]
3. **[Need]** (resonance: 3/4) — [rationale]

---
*Generated by /auto-req-output on YYYY-MM-DD*
```

Print: "Requirements report → reports/YYYY-MM-DD-requirements.md"

## Step 3: Generate Skill Drafts

For the top 3 needs from the priority ranking, dispatch 3 subagents **in parallel**. Each gets one need. Use this prompt for each (fill in [NEED], [RESONANCE], [RATIONALE], [QUOTE] with actual values):

```
SKILL DRAFT GENERATOR

Write a Claude Code skill file for this identified user need.

NEED: [NEED]
RESONANCE: [RESONANCE] developer personas independently raised this
RATIONALE: [RATIONALE]
REPRESENTATIVE QUOTE: "[QUOTE]"

Output ONLY a skill file in this exact format (including the --- delimiters). No explanation.

---
name: [kebab-case-skill-name]
description: [One specific sentence: when to invoke this skill and what it does. Must be specific enough Claude knows exactly when to use it.]
---

# [Skill Title]

## When This Skill Applies
[1-2 sentences describing the exact trigger situation]

## What To Do

[Step-by-step instructions for Claude Code. Be concrete. Include:
- What information to gather first
- What actions to take
- What output to produce]

## Example Usage

User: "[example user message that would trigger this skill]"

Claude should: [what Claude does]

Output: [what the user receives]
```

Save each draft as `skills/drafts/YYYY-MM-DD-[skill-name].md`.

