# 标题 vs 现实仪表板

## 功能说明

当恐慌性市场标题出现在主流新闻中时，快速展示标题叙事（散户恐惧方向）vs 机构买家实际行为（鲸鱼钱包、ETF 资金流、企业国库购入）的对比，产出简单的持有/等待/谨慎信号。输出针对零金融专业知识的用户优化，30 秒可读。

## 用户画像

**小戴（加密新手，月薪族）**
- **角色**: 普通上班族，在 Cash App 上持有 $200–$500 的 BTC，无金融背景
- **日常工作流**: 下班后刷 Twitter/Reddit，看到恐慌性标题就担心，靠直觉决定要不要卖
- **工具栈**: Cash App、Reddit、Twitter，不使用任何专业分析工具
- **痛点**: 无法区分"这次崩盘是真的"还是"媒体在制造恐慌"。BTC 跌 3% 他就开始焦虑，但他的持仓只有 $300，跌 10% 也只亏 $30。他需要有人用大白话告诉他该不该担心，以及大买家在做什么。

**Bob（散户量化交易者）**
- **角色**: 业余量化，用 Python + TradingView 做加密和美股，组合规模 $5,000–$30,000
- **日常工作流**: 早上看宏观新闻，白天看 CT（Crypto Twitter），用 Claude Code 写回测；遇到大标题时手动查鲸鱼数据和资金费率，耗时 20–30 分钟
- **工具栈**: TradingView、Binance、Python、Claude Code
- **痛点**: 宏观媒体（Bloomberg/CNBC）和 CT 经常讲相反的故事。2026 年 3 月，CT 喊 ETF 叙事利好、机构买入，但宏观账号警告美元走强。他没有工具量化这种分歧，按照 CT 信号全仓操作，结果遭遇 12% 回调。他需要 30 秒内看到"标题 vs 机构实际行为"的对比结论，而不是在多个工具间切换。

## 用户故事

- 作为**小戴**，我想在看到恐慌标题后用大白话查清楚"鲸鱼在买还是在卖"，这样即使我对加密一无所知，也能在 30 秒内决定是否需要动我的 $300 持仓。
- 作为**Bob**，我想在宏观标题出现时立即对比"标题叙事 vs 机构实际行为"，这样我不用再花 20–30 分钟手动跨平台拼图，也不会再因为 CT 和 Bloomberg 讲不同故事而判断错误。

## 适用场景

用户看到一个市场标题（CNBC、Bloomberg、Reddit、Twitter 等），感到焦虑、不确定或被迫行动，想在做小到中等规模组合的决策前做快速理性检查。

**触发短语**: "该不该卖我的 BTC", "这次崩盘是真的吗", "标题说崩盘/暴跌/恐慌", "鲸鱼在买还是在卖", "机构在做什么", "标题 vs 现实", "该不该担心这个标题", "恐惧有根据吗", "smart money vs scared money", "CNBC 说某个吓人的东西——是真的吗", "某个协议/平台出事了，我的 BTC/ETH 会受影响吗", "这个坏消息跟我有没有关系"

**注意**：当用户担心的标题涉及与其持仓无直接关联的资产（如 DeFi 协议被黑但用户持有的是 BTC），第一步需先判断事件与用户持仓的相关性，并在输出中明确标注"与你的持仓直接相关 / 间接相关 / 无直接关联"。

## 输入参数

| 参数 | 必须 | 默认值 | 说明 |
|------|------|--------|------|
| 标题/事件 | 是 | - | 用户看到的标题或事件 |
| 用户持仓 | 否 | "小额 BTC 持有者" | 持有什么、大约多少 |

如用户未提供持仓，默认为"小额 BTC 持有者"并注明建议为通用型。

## 执行步骤

### 第一步：捕获标题上下文并分类恐慌类型

从用户消息提取: 标题/事件内容、用户持仓（资产和金额）。

**恐慌类型分类（必须执行）：** 根据标题内容判断恐慌来源，后续分析逻辑因类型而异：

| 类型 | 识别特征 | 机构行为解读差异 |
|------|---------|----------------|
| **清算驱动型** | 标题提及"爆仓"、"liquidation cascade"、大额清算金额、某交易所/协议穿仓 | 机构不动 = 等待强制卖盘耗尽（偏多），机构也在卖 = 系统性风险 |
| **叙事/地缘驱动型** | 标题涉及战争、制裁、监管、政策、宏观事件 | 机构不动 = 也在评估中（中性），机构买入 = 押注过度恐慌 |
| **操纵/单一行为者型** | 价格大幅波动但仅出现在单一平台或单一大额交易触发 | 信号自动降为低可信度，需验证多平台是否跟随 |
| **传染恐慌型** | A 资产出事但用户担心自己的 B 资产 | 先判断关联度再出信号，无关联则直接"与你无关" |
| **信号分歧型** | 不同信息源讲相反故事（如 CT 看多但宏观数据看空，或 ETF 流入但链上鲸鱼流出） | 不存在单一方向可信赖，必须比较各信源的数据硬度：链上数据 > 机构持仓 > 社交媒体叙事。输出须明确标注哪方证据更硬、信号默认为"等待" |

将分类结果标注在输出中，后续步骤的因果链分析和信号判定须参考此分类。

### 第二步：获取标题端——主流情绪

```
ant_market_sentiment(query_type="topics_list")
```

找到与用户标题相关的话题，然后:

```
ant_market_sentiment(query_type="topic_posts", topic="<relevant_topic>")
```

提取: headline_sentiment, fear_level, dominant_narrative, post_count

```
ant_market_indicators(category="sentiment")
```

提取: fear_greed_score (0-100)

### 第三步：获取现实端——机构买家实际行为

```
ant_smart_money(query_type="whale_movements", asset="BTC")
```

提取: whale_net_flow, whale_buy_count vs whale_sell_count, notable_buyers

**单一行为者集中度检测（必须执行）：**
检查 whale_movements 返回的数据：
- 如果净流向主要由 1-2 个地址驱动（单笔交易占总净流 > 50%），标注 `信号集中度: 高（单一行为者主导）`
- 如果买/卖方各有 5+ 个独立地址，标注 `信号集中度: 低（广泛参与）`
- 高集中度时，该数据源的权重在后续信号判定中自动降级（不能单独支撑 HOLD 或 CAUTION 信号）

同时调用:
```
ant_fund_flow(query_type="exchange_whale_ratio", asset="btc", exchange="binance")
```
提取 whale_ratio：该指标衡量大额交易占总交易的比例。如果 whale_ratio 突然飙升但方向与整体市场不一致，进一步确认单一行为者操纵风险。

**预测市场/外部平台信号的间接验证（当标题涉及预测市场赔率变动时必须执行）：**
当用户标题涉及预测市场（Polymarket、Kalshi 等）赔率大幅变动时，MCP 无法直接获取预测市场订单簿数据。此时用间接方法检测单一行为者操纵：
1. 检查标的资产的链上 whale_movements 和交易所 whale_ratio 是否有对应量级的变动
2. 检查 ant_futures_market_structure（OI、funding rate）是否有对应方向的变化
3. 如果预测市场赔率大幅变动（>10pp）但链上/衍生品市场均无对应反应 → 标注 `预测市场信号未经链上确认: 高概率单一行为者操纵或流动性极薄`，信号自动降为"等待"
4. 在输出中明确写出："预测市场赔率变动未被链上数据印证——可能是单一大额下注而非市场共识"

```
ant_macro_economics(query_type="overview")
```

提取: institutional_positioning, macro_backdrop

可选:
```
ant_market_indicators(category="etf_flows")
```

### 第四步：计算标题 vs 现实差距

对比两端:
- **一致性**: 标题和机构行为是否讲同一个故事？（一致 / 混合 / 分歧 / 强烈分歧）
- **分歧方向**: 哪一方更看空？
- **信念强度**: 机构信号有多强？（弱 / 中等 / 强）

**因果链分析（必须执行，不可跳过）：**
在确定信号之前，必须完成以下 3 层推理并写入输出：
1. **机制层（为什么）**: 机构为什么在这样做？可能的机制包括：折价基差交易（basis trade）、ETF 再平衡强制买入、OTC 大宗定位、恐慌流动性收割、跟随宏观政策转向。从 ant_smart_money 和 ant_macro_economics 数据推断最可能的机制。
2. **后果层（所以呢）**: 该机制对散户持有者意味着什么？例如：机构在折价买入 → 他们预期价格回升 → 散户此时卖出等于把折价筹码送给机构。
3. **逆转条件层（什么时候失效）**: 明确写出 1-2 个具体的、可观测的条件，如果这些条件发生，则当前判断逆转。例如："如果 24 小时内积累的鲸鱼钱包开始分销，或 ETF 连续 2 天净流出超 $200M，则 HOLD 信号失效，应转为 CAUTION。"

信号判定:

| 标题 | 机构 | 信号 |
|------|------|------|
| 看空 | 积累中 | 持有——标题比现实更嘈杂 |
| 看空 | 也在卖 | 谨慎——恐惧可能有根据 |
| 看多 | 分销中 | 谨慎——乐观可能缺乏大资金支撑 |
| 看多 | 也在买 | 持有/考虑加仓——双方一致 |
| 混合 | 任何 | 等待——画面不清楚 |
| 任何 | 混合（买卖并存）| 等待——机构内部也没有共识，不宜仓促行动 |

> **机构信号"混合"的判断标准**: 鲸鱼买家和卖家数量接近（例如 10 个大买家 vs 8 个大卖家），或净流入接近零，或同一周内出现过大额买入和大额卖出。此时信念强度评为"弱"，无论标题方向如何均输出"等待"信号。

### 第五步：生成输出

输出必须零金融术语，30 秒可读。

```
========================================
  标题 vs 现实核查
  [日期]
========================================

标题端

  帖子在说什么: [1-2 句通俗语言]

现实端

  大买家比例: [X 个大买家中有 Y 个本周加仓]

差距
  标题说: [一个短语]
  大资金说: [一个短语]
  差距: [一致 / 混合 / 分歧 / 强烈分歧]

  大买家这样做的原因: [用通俗语言解释机制，
    如"他们在趁恐慌打折买入"或"跟着政策方向走"]

  你的信号:  [持有 / 等待 / 谨慎]
```

## 使用示例

**用户**: "CNBC 刚说 'Crypto selloff deepens'，我的 BTC 跌了 6%。我在 Cash App 上有 $300。该卖吗？"

→ 执行全部步骤。如果机构在积累（如 Strategy 购入 $330M BTC 低于成本基础、ETF 净流入 $120M）而恐惧贪婪指数 22，输出信号"持有"，解释标题比现实更嘈杂，$300 仓位即使再跌 10% 也只损失 $30。

**用户**: "机构也在卖吗？我一直看到看空标题，担心这次不一样。"

→ 重新获取最新数据。如果机构行为确实转为净卖出，信号变为"谨慎"，诚实说明标题和大买家这次方向一致。

## 反模式

1. **不要使用金融术语而不翻译。** 永远不说"资金费率为负"而不立即解释含义。
2. **不要给精确价格目标。** 这是方向性理性检查，不是价格预测。
3. **不要假设用户应该总是持有。** 标题和机构都看空时，要说实话。
4. **不要信息过载。** 输出 30 秒可读。深度分析用 /smart-money-spread。
5. **不要忽略持仓规模。** $300 仓位和 $30,000 仓位面对同样标题需要不同建议。跌 10% 在 $300 是 $30——要明确说出来。
6. **不要把鲸鱼数据当作确定性。** 用"历史上"、"倾向于"而不是"会"或"总是"。
7. **不要跳过机构端。** 只展示标题情绪而不检查大买家行为 = 重现了这个技能要解决的问题。
8. **不要要求 API 密钥或配置。** 用户只需输入一个标题和持仓描述。
