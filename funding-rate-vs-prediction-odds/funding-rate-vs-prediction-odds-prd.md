# Funding Rate vs Prediction Market Odds - 需求文档 (PRD)

## 需求概述

在用户进入杠杆加密仓位前，需要一个工具将永续合约资金费率（Binance、Bybit、dYdX）与预测市场事件赔率（Polymarket、Kalshi）叠加对比，自动检测两者之间的背离。目前这两类数据分别存在于不同的浏览器标签页中，交易者（如 Carol）因为串行处理这些信号而遭受了实际亏损——检查标签页一和标签页二之间的时间差恰恰是最大回撤发生的地方。

## 用户画像

### Carol -- 全栈 SaaS 开发者 / 跨资产交易者

- **角色**: 交易 MSTR 和 BTC，跨股票和加密现货，每日使用 Claude Code 做后端逻辑和 API 集成
- **工具栈**: Next.js, Supabase, Tailwind, TypeScript, Claude Code（每日）, Binance, CoinGlass, Polymarket, 券商终端, Excel
- **痛点**: Carol 的痛点不是缺少数据，而是缺少综合分析：她同时拥有资金费率和预测市场赔率，但串行处理它们。她最大的回撤发生在查看 CoinGlass（资金费率正向=看多）和查看 Polymarket（鹰派赔率从 35% 飙升至 62%）之间的时间差。她因仅基于资金费率做多 MSTR 而亏损 4%+。

### Alice -- DeFi 全栈独立开发者 / 链上分析师

- **角色**: 写 Solidity 合约，做链上数据分析，维护跨链数据管道
- **工具栈**: Solidity, TypeScript, Python, Hardhat, Dune Analytics, The Graph, Arkham, CoinGlass, Claude Code（每日）
- **痛点**: Alice 已经程序化监控资金费率，将预测市场赔率视为另一个可观察数据源，应该被导入同一个跨源综合分析层。她在 Round 2 的观察——"追踪分析师观点是追踪滞后指标，而领先指标已经在链上和期货数据中公开可见"——直接支持了资金费率和预测市场赔率属于同一视图的前提。

## 用户故事

1. **作为 Carol**，一个持有 MSTR 和 BTC 的跨资产交易者，我想在进入仓位前并排看到资金费率和宏观事件赔率，以便不会被衍生品市场忽略的宏观催化剂打个措手不及。

2. **作为 Alice**，一个构建链上分析工具的 DeFi 开发者，我想获得衍生品持仓和事件概率之间的结构化背离信号，以便将其纳入我的鲸鱼资金流分析管道作为额外上下文层。

3. **作为加密交易者在美股开盘前检查仓位时**，我想知道隔夜资金费率变动是否与即将到来的宏观事件的预测市场赔率一致，以便决定动量交易是否可以安全持有或需要对冲。

4. **作为加密交易者 SaaS 仪表板的构建者**，我想要一个可重复的方法来计算资金费率 vs 赔率背离，以便将其产品化为自己工具中的一个功能。

## 需求边界

### 做什么
- 叠加永续合约资金费率与预测市场宏观事件赔率
- 计算量化背离分数并分配体制标签
- 提供时间敏感性评估（距离催化事件的天数）
- 标注资金费率极端值
- 识别哪一方可能定价错误

### 不做什么
- 不预测价格方向
- 不在用户仅询问资金费率而无宏观事件参考时触发
- 不在用户仅询问预测市场赔率而无衍生品上下文时触发
- 不提供仓位大小建议
- 不替代深度宏观研究
